1) Designed and implemented a module for handling domain-specific data, encompassing both server-side and client-side functionality. The module manages the retrieval, processing, transmission, and storage of dynamic hierarchical structures, and performs the following:

  • •  extracting flat data from a relational database;
  • •  converting it into a hierarchical structure according to the relationship schema;
  • •  processing the data programmatically in accordance with business logic;
  • •  transforming it into a format suitable for the client side;
  • •  sending the data to the client;
  • •  rendering it on the client within a dynamic tree component, equipped with GUI functionality for editing, deleting, adding, and moving nodes;
  • •  after modifications, sending a summary about changes to the server;
  • •  on the server, converting the modifications into the format required by server-side logic with accordance to the info received, updating the database, and synchronizing related programmatical objects.

As part of this work, a custom data structure was invented and paired with an optimized algorithm that reduces traversal complexity from the estimated O(N + T + R) [where N is the number of data nodes, T is the number of tables, and R is the number of relationships defining the hierarchy] to O(N) that is significantly more economical, especially when data is retrieved portionally and T + R may exceed N.
